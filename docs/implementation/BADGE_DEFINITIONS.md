## Implementation Notes

### Badge Check Triggers
1. **After workout completion** → Check strength count, volume, PR badges
2. **After run completion** → Check running distance, speed, elevation badges
3. **Daily job** → Check streak badges, weekly consistency
4. **On PR set** → Check specific lift milestones (plate badges)

### Notification Strategy
- Show badge unlock animation immediately
- Send push notification for Gold/Platinum badges
- Add to activity feed for social sharing

### Database Schema
```sql
-- Badge definitions table (see schema.sql)
CREATE TABLE badge_definitions (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  description TEXT,
  badge_type TEXT NOT NULL, -- 'strength', 'running', 'streak', 'hybrid'
  icon_url TEXT,
  criteria JSONB, -- { type: 'workout_count', value: 100 }
  tier TEXT DEFAULT 'bronze',
  created_at TIMESTAMP DEFAULT NOW()
);

-- User badges table
CREATE TABLE user_badges (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  badge_id TEXT REFERENCES badge_definitions(id),
  earned_at TIMESTAMP DEFAULT NOW(),
  metadata JSONB -- { workout_id, run_id, etc. }
);
```
